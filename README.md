# volkancicek.com

Personal website and blog built with [Pelican](https://getpelican.com/).

## Quick Start

### Prerequisites

- Python 3.10+
- [uv](https://docs.astral.sh/uv/) package manager

### Setup

```bash
# Create virtual environment and install dependencies
uv venv
uv pip install -r requirements.txt
```

### Development

```bash
# Start development server with live reload
uv run pelican --listen --autoreload

# Or using invoke tasks
uv run invoke livereload
```

The site will be available at `http://localhost:8000`.

### Build

```bash
# Build the site
uv run pelican content

# Build for production
uv run pelican content -s publishconf.py
```

## Infrastructure

> Live infra reference. Future contributors and agents should work from this, not guesswork.

### Domain & DNS

- **Domain:** `volkancicek.com` (apex). All traffic canonicalised to apex.
- **DNS provider:** Cloudflare (zone uses Cloudflare nameservers `cameron.ns.cloudflare.com`, `teagan.ns.cloudflare.com`).
- **Registrar:** external (in-zone NS records `*.dnsenable.com` are vestigial leftovers from previous setup, ignored externally since registrar points to Cloudflare NS).

DNS records (Cloudflare → DNS → Records):

| Type | Name | Content | Proxy | Purpose |
|------|------|---------|-------|---------|
| CNAME | `volkancicek.com` | `volkancicek.pages.dev` | Proxied | Apex → Cloudflare Pages origin |
| CNAME | `www` | `volkancicek.com` | Proxied | Caught by Redirect Rule → apex (see below) |
| TXT | `volkancicek.com` | `v=spf1 -all` | DNS only | SPF: no server is authorised to send mail from this domain |
| TXT | `_dmarc` | `v=DMARC1; p=reject; rua=mailto:volkanciicek@gmail.com; ruf=mailto:...; fo=1` | DNS only | DMARC: reject spoofed mail, reports to owner |
| TXT | `volkancicek.com` | `google-site-verification=...` | DNS only | Auto-added by Search Console via Cloudflare integration; **do not delete** |

No MX record (intentional — this domain does not receive email; owner uses `volkanciicek@gmail.com`).

### Hosting — Cloudflare Pages

- **Project name:** `volkancicek` (origin at `volkancicek.pages.dev`)
- **Connected repo:** [`volkanncicek/volkancicek.com`](https://github.com/volkanncicek/volkancicek.com) (branch: `main`)
- **Auto-deploy:** every push to `main` triggers a build via Cloudflare Pages' GitHub integration (no GitHub Actions involved on this repo).
- **Build settings:**

| Setting | Value |
|---------|-------|
| Build command | `pelican content -s publishconf.py` |
| Build output directory | `output` |
| Root directory | `/` |

### Edge rules (Cloudflare)

- **Redirect Rule — "WWW to root"**: any request matching hostname `www.volkancicek.com` returns `301` to `https://volkancicek.com/${path}`. Runs at edge before Pages, fully bypasses the origin so `www` never has to be configured as a Pages custom domain.

### Predecessor site (kept alive as a redirect)

- **Repo:** [`volkanncicek/volkanncicek.github.io`](https://github.com/volkanncicek/volkanncicek.github.io) (branch: `master`)
- **Hosting:** GitHub Pages (workflow `.github/workflows/static.yml`)
- **Behaviour:** `index.html` + `404.html` are tiny stubs that send all traffic to `https://volkancicek.com/` via `<meta http-equiv="refresh">`, JS `location.replace`, and `<link rel="canonical">`. Zero ongoing maintenance.
- **Why kept:** Preserves old links from CV / LinkedIn / search index without losing SEO equity.

### SEO & indexing

- **Google Search Console:** Domain property for `volkancicek.com`, verified via Cloudflare DNS integration (auto-added TXT). Sitemap submitted: `https://volkancicek.com/sitemap.xml`.
- **Bing Webmaster Tools:** Property imported from Search Console (no separate verification needed).
- **Sitemap:** generated by Pelican `sitemap` plugin (configured in `pelicanconf.py`). Auto-includes all posts, pages, tags, categories.
- **Optional config hooks** in `pelicanconf.py` (currently empty, available if Domain verification ever needs a fallback meta tag):
  - `GOOGLE_SITE_VERIFICATION`
  - `BING_SITE_VERIFICATION`

### Local development → production flow

1. Edit content / templates locally.
2. `uv run pelican --listen --autoreload` for live preview at `http://localhost:8000`.
3. Commit + push to `main`.
4. Cloudflare Pages auto-builds with `pelican content -s publishconf.py` and deploys.
5. Cache propagates globally in ~30–90 seconds.
